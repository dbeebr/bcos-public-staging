# BCOS Manifesto

BCOS stands for **Base Context Operating System**.

It is a coordination layer for distributed work between humans, AI agents, teams, tools, and time.

## Chat History Is Not Enough

Modern work now happens across conversations, documents, repositories, issue trackers, meetings, agents, and time zones. The context exists, but it is scattered. A chat transcript may contain useful clues, yet it is not a reliable operating surface.

Chat history is chronological. Work is structural.

Chat history records what was said. Work needs to know what is true, what is proposed, what is decided, what is blocked, what is safe to do, and what evidence proves it.

BCOS treats conversation as source material, not as truth.

## Context Must Become Operational Capital

Context is not a side effect of work. It is an asset that determines whether future work can continue cleanly.

When context is unmanaged, every new person or agent pays the same orientation cost. Decisions get re-litigated. Tasks become vague. Handoffs become folklore. Evidence disappears into screenshots and memory.

BCOS turns context into operational capital: structured, durable, reviewable, and reusable.

The goal is not to document everything. The goal is to preserve the smallest sufficient truth that lets work continue.

## Tasks Need Executable Contracts

A task is not just a request. A serious task needs a contract:

- what is being attempted;
- what is allowed;
- what is forbidden;
- what must be read first;
- what evidence must exist before completion can be claimed;
- who or what must approve irreversible steps.

This matters more when AI agents participate. Agents are powerful, literal, and fast. They need bounded contracts, not vibes.

BCOS tasks are designed to be executable by humans and agents without requiring a full private memory dump.

## Evidence Matters More Than Claims

"Done" is not a feeling.

For distributed work, completion must be backed by evidence: a commit, a review, a validation report, a reproduced result, a source link, or another durable proof.

BCOS prefers evidence over confidence. Claims can guide attention; evidence earns trust.

This does not make work bureaucratic. It makes work transmissible.

## Human Gates Matter

Automation should not erase judgment.

Some decisions are cheap and reversible. Some decisions are expensive, public, private, legal, architectural, or hard to undo. BCOS separates ordinary execution from Human Gates so that work can move quickly without pretending all choices are equal.

A Human Gate is a deliberate pause for accountable human judgment.

It is how BCOS keeps speed and responsibility in the same system.

## Distributed Human-AI Work Needs Shared Structure

Human teams already struggle with handoffs, context drift, and decision memory. AI agents amplify the problem because they can produce large amounts of work from partial context.

The answer is not more chat, more dashboards, or more dashboards about chat.

The answer is shared structure:

- context indexes;
- source-of-truth files;
- typed tasks;
- explicit decisions;
- handoffs;
- proof records;
- redaction boundaries;
- human approval gates.

BCOS is a file-native attempt to make that structure simple enough to use and strict enough to trust.

## The Public Promise

BCOS v0.1 should give people a reusable way to coordinate work that outlives the conversation that started it.

It should help a team or solo builder answer:

- What are we trying to do?
- What is already decided?
- What is merely proposed?
- What must not be exposed?
- What can an agent safely execute?
- What evidence proves the work happened?
- Where should the next person start?

BCOS is not a replacement for human judgment. It is a structure that helps judgment travel.
