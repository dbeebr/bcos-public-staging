# Contributing to BCOS

BCOS v0.1 is experimental. Contributions are welcome within the scope below.

## What We Welcome

- Clearer wording in docs and templates
- Safer or more useful templates
- Neutral, privacy-respecting examples
- Missing definitions or explanations
- Better onboarding and first-run guidance
- Redaction and privacy improvements
- Bug reports in examples or templates

## What Requires Maintainer Review

Changes to any of the following require maintainer review and, for significant
changes, an explicit Human Gate decision:

- Core primitives in `docs/core-model.md`
- Public naming or branding
- License or governance model
- Security posture or disclosure policy
- Architecture decisions
- Runtime code (not included in v0.1 docs-only release)

## How to Contribute

1. Open an issue to describe the change and get early feedback.
2. Fork the repository and create a branch from `main`.
3. Make your changes with clear commit messages.
4. Open a pull request against `main`.
5. Describe what changed and why in the PR description.

## Pull Request Scope

v0.1 accepts pull requests for:

- Documentation (`docs/`)
- Templates (`templates/`)
- Neutral examples (`examples/`)

Pull requests that change core model primitives, architecture, license, or
governance will be reviewed by a maintainer and may require a Human Gate decision
before merging.

## Code of Conduct

All contributors are expected to follow the [Code of Conduct](CODE_OF_CONDUCT.md).

## Issue Reporting

Use GitHub Issues to report unclear documentation, template errors, privacy
concerns, or missing definitions. Please include enough context for a maintainer
to reproduce or understand the issue.

## Status

BCOS v0.1 is a working standard, not a finished product. Feedback on usability,
correctness, and privacy is especially valuable at this stage.
