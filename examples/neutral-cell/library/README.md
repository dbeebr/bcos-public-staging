# Library — Neutral Example Cell

This folder shows what a nascent Library looks like in a BCOS Cell.

## What Is Here

After operating for several weeks, this example Cell has distilled one proven
pattern from its work:

### Proven Task Pattern: Documentation Review

Through completing `tasks/example-task.md` and similar documentation tasks, the
Cell found that documentation tasks work best when structured as:

1. Define the audience and scope before writing.
2. List required source context explicitly in the task.
3. Require a Human Gate before any public-facing publication.
4. Record the final commit SHA as proof of completion.

This pattern was used three times and produced clean handoffs each time. It is
now a Library entry that future documentation tasks in this Cell can start from.

## What Is Not Here

This Library does not contain:

- Raw session logs or chat history
- Draft templates that have not been tested
- Every artifact the Cell has produced

A Library is curated. Only patterns that have been validated through use belong here.

## How This Grows

As the Cell operates longer, more patterns will be distilled:

- Decision patterns for recurring choice types
- Handoff formats that reliably transfer context
- Playbooks for common workflows

Each entry earns its place by being used and proven, not just proposed.
