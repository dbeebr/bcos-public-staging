# Support

## BCOS v0.1 Support Expectations

BCOS v0.1 is an **experimental** release. It is a working standard, not a
finished product or a supported commercial service.

Support is best-effort and community-driven.

## How To Get Help

- **GitHub Issues:** The primary channel for questions, bug reports, and
  feedback. Open an issue with as much context as possible.
- **Documentation:** Start with `docs/manifesto.md`, `docs/core-model.md`, and
  `docs/public-private-boundary.md`.
- **Templates and Examples:** The `templates/` and `examples/neutral-cell/`
  directories are the practical starting point.

## What We Can Help With

- Clarifying how BCOS concepts apply to a use case
- Understanding template fields and their purpose
- Improving documentation that is unclear
- Reporting errors in examples or templates

## What Is Out of Scope for v0.1

- Enterprise support or SLA commitments — none are offered at v0.1
- Runtime implementation support — v0.1 does not include runtime code
- Private operating system consultation
- Bespoke BCOS customization or consulting

## Response Time

This is an experimental community release. Response times are not guaranteed.
Maintainers will make reasonable efforts to review issues and pull requests.

## Status

BCOS v0.1 is experimental. Expect rough edges, missing pieces, and ongoing
refinement. Feedback that improves the standard for everyone is especially
welcome.
